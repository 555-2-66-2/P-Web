specificity

Specificity is the algorithm that the browser uses to decide which property value is applied to an element. If multiple style blocks have different selectors that configure the same property with different values and target the same element, specificity decides the property value that gets applied to the element. Specificity is basically a measure of how specific a selector's selection will be:

  • An element selector is less specific; it will select all elements of that type that appear on a page, so it has less weight. Pseudo-element selectors have the same specificity as regular element selectors.
  • A class selector is more specific; it will select only the elements on a page that have a specific class attribute value, so it has more weight. Attribute selectors and pseudo-classes have the same weight as a class.
  • Attribute substring matching selectors lebih lemah dari Attribute presence and value selectors

contoh: terdapat selector body yg memiliki properti color dgn value blue dan jg terdapat selector span yg memiliki selector color dgn value black. semua text di dalam body berwarna blue kecuali text di dalam span yg akan berwarna black

Properti CSS dapat dibagi menjadi dua jenis: inherited (diwarisi) dan non-inherited (tidak diwarisi). Properti inherited akan diwarisi oleh elemen anak dari elemen induknya, sedangkan properti non-inherited tidak diwarisi dan harus diatur secara eksplisit untuk setiap elemen.

margin, padding, border, background, color, font-family, font-size, font-weight, line-height, text-align, display, position, float, width, height adalah non-inherited properties

lima universal property values untuk mengendalikan pewarisan (inheritance) dalam CSS:

  1. inherit: Memungkinkan elemen anak untuk mewarisi nilai properti dari elemen induknya. Jika nilai properti diatur pada elemen induk, elemen anak akan mewarisi nilai tersebut. Jika nilai properti tidak diatur pada elemen induk, elemen anak akan menggunakan nilai default (initial) untuk properti tersebut.
  2. initial: Mengatur properti kembali ke nilai awal (default). Elemen anak akan menggunakan nilai awal dari properti tersebut, tanpa memperhitungkan nilai yang diatur pada elemen induk.
  3. unset: Mengatur properti ke nilai yang diwarisi jika properti tersebut dapat diwariskan, dan ke nilai awal jika properti tersebut tidak dapat diwariskan. Jika nilai properti diatur pada elemen anak, maka unset akan mengembalikan properti tersebut ke nilai diwarisi dari elemen induk. Jika nilai properti tidak diatur pada elemen anak, maka unset akan berperilaku seperti initial, menggunakan nilai default untuk properti tersebut.
  4. revert: Mengatur properti kembali ke nilai sebelumnya sebelum ada penugasan nilai apapun. Jika nilai properti diatur pada elemen anak, maka revert akan mengembalikan properti tersebut ke nilai sebelumnya sebelum ada penugasan. Jika nilai properti tidak diatur pada elemen anak, maka revert akan berperilaku seperti inherit, mencari nilai yang diwarisi dari elemen induk.
  5. revert-layer: Mirip dengan revert, namun nilainya lebih kuat. Biasanya digunakan dalam kasus ketika ada konflik nilai dengan cascading dan ingin menggunakan nilai yang ditentukan oleh revert. Saat menggunakan revert-layer, nilai-nilai yang diterapkan melalui revert tidak akan mempengaruhi nilai properti yang ditetapkan setelahnya, kecuali ada konflik lebih lanjut.

Jika beberapa layer telah dibuat sebelum pernyataan di atas, selama layer dengan nama-nama tersebut belum ada, maka ketiga layer tersebut akan dibuat dan ditambahkan pada akhir daftar layer yang sudah ada. block and inline boxes

outer display type

  • block
    the box will break onto a new line. padding, margin, dan border akan menyebabkan elemen lainnya terdorong menjauhi kotak. Jika lebar tidak ditentukan, kotak akan melebar ke arah sejajar untuk mengisi ruang yang tersedia dalam wadahnya. Pada sebagian besar kasus, kotak akan menjadi selebar wadahnya, mengisi 100% ruang yang tersedia.
    contoh HTML elements yg menggunakan block as their default outer display type ada h1-h6, div, ul, ol, li, table, form, header, footer, article, nav, aside, p, dsb.
  • inline
    The box will not break onto a new line. The width and height properties will not apply. Top and bottom padding, margins, and borders will apply but will not cause other inline boxes to move away from the box. Left and right padding, margins, and borders will apply and will cause other inline boxes to move away from the box.
    contoh HTML elements yg menggunakan inline as their default outer display type ada span, a, strong, b, em, i, br, img, input, label, select, cite, q, code, abbr, sup, sub, dsb.

inner display type

bisa diubah dengan style display: block/inline/inline-block/flex/grid/dsb

background attachment

  • scroll
    causes the element's background to scroll when the page is scrolled. If the element content is scrolled, the background does not move. In effect, the background is fixed to the same position on the page, so it scrolls as the page scrolls.
  • fixed
    causes an element's background to be fixed to the viewport so that it doesn't scroll when the page or element content is scrolled. It will always remain in the same position on the screen.
  • local
    fixes the background to the element it is set on, so when you scroll the element, the background scrolls with it.

writing-mode

  • horizontal-tb: Top-to-bottom block flow direction. Sentences run horizontally.
  • vertical-rl: Right-to-left block flow direction. Sentences run vertically.
  • vertical-lr: Left-to-right block flow direction. Sentences run vertically.

overflow

digunakan ketika Anda ingin mengontrol perilaku tampilan konten yang melebihi ukuran kotak elemen. biasanya digunakan pada elemen dengan ukuran tetap (fixed size) atau elemen yang memiliki ukuran terbatas dalam tata letak.

  • overflow: hidden. Konten yang melebihi ukuran elemen akan disembunyikan dan tidak akan terlihat di luar batas elemen. Bagian yang melebihi akan "terpotong" atau dianggap tidak ada.
  • overflow: scroll. Browsers with visible scrollbars will always display them—even if there is not enough content to overflow.
  • overflow-y/x: scroll. scrollbarnya di sumbu y/x
  • overflow: auto. If you only want scrollbars to appear when there is more content than can fit in the box.
  • overflow: visible. Nilai ini merupakan nilai default. Konten yang melebihi ukuran elemen akan tampil di luar batas elemen, dan mungkin tumpang tindih dengan elemen di sekitarnya.
  • overflow: overlay. Konten yang melebihi ukuran elemen akan disembunyikan, tetapi scrollbar hanya akan muncul jika pengguna mencoba scroll konten tersebut.
